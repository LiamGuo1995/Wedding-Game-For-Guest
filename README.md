# 接喜糖挑战 Wedding Candy Game

一个适合婚礼现场扫码游玩的 30 秒手机网页小游戏。来宾填写姓名后开始游戏，每人最多 3 次机会，系统记录最高分并生成排行榜。项目保留微信登录接入点，也支持先用姓名 + 设备标识的方式落地运行。

## 功能

- 手机网页游戏，支持 iOS Safari、安卓浏览器、微信内置浏览器
- 30 秒接喜糖玩法，包含喜糖、红包、囍字礼物、扣分酒杯
- 难度随时间递增，支持连击加成，便于拉开分数
- 来宾进入前填写姓名或称呼
- 每人最多 3 次机会
- 实时排行榜
- 管理员后台查看参与人数、完成局数和榜单
- 婚礼信息可配置：新人姓名、日期、地点、奖励文案、主题色
- 预留微信 OAuth 接口，生产环境可替换为公众号/服务号登录

## 本地运行

```bash
cp config.example.json config.json
cp data/db.example.json data/db.json
APP_SECRET="replace-with-random-secret" ADMIN_PASSWORD="your-admin-password" node server.js
```

打开：

- 游戏页：http://localhost:3000
- 管理后台：http://localhost:3000/admin

默认后台密码是 `admin123`，正式使用前必须通过 `ADMIN_PASSWORD` 环境变量修改。

## GitHub Pages 试玩版

仓库包含 GitHub Pages 自动部署配置。推送到 `main` 后，会把 `public/` 目录部署为静态试玩版。

静态试玩版可以直接打开、填写姓名、玩游戏、查看本机排行榜；但它没有真正的服务端数据库，因此排行榜只保存在当前手机或浏览器里，不能作为婚礼现场全场排名使用。

婚礼现场正式使用时，请部署 Node 服务版本，这样所有来宾分数才会进入同一个排行榜，管理员后台也才能看到全场数据。

## 现场使用建议

- 把游戏链接生成二维码，放在签到区、合影区和桌牌上
- 现场安排 1 个亲友保管后台密码，负责看榜和发奖
- 如果农村网络不稳，建议提前测试宴席地点的 4G/5G 和 Wi-Fi
- 最终颁奖前，后台刷新排行榜即可

## 微信登录接入说明

当前版本默认使用“姓名 + 浏览器设备标识 + cookie”限制 3 次，适合先做婚礼现场 MVP。

如果要作为商业产品卖给别人，建议接入微信网页授权：

1. 准备已认证服务号或开放平台能力
2. 准备已备案域名和 HTTPS
3. 在 `/auth/wechat/start` 跳转微信 OAuth
4. 在 `/auth/wechat/callback` 用 `code` 换取 `openid`
5. 用 `openid` 替代当前的设备标识作为玩家唯一身份

注意：微信网页授权、支付、备案、隐私政策、用户数据存储都属于正式商用前必须处理的事项。

## 商业化后续路线

- 数据库换成 PostgreSQL、MySQL 或云开发数据库
- 支持多婚礼/多租户，每个客户一个独立配置和排行榜
- 后台增加导出 Excel、清空测试数据、奖项设置
- 增加素材上传：新人照片、背景图、喜糖样式
- 增加防作弊：服务端生成掉落序列、回放校验、异常分数拦截
- 增加微信登录、小程序版本或公众号网页版本

## 部署

这个项目是纯 Node.js 服务，可以部署到常见平台：

- 阿里云 / 腾讯云 / 华为云轻量服务器
- Railway、Render、Fly.io 等 Node 托管平台
- Docker 化后部署到任意容器平台

中国大陆现场使用建议选择大陆服务器和已备案域名，否则微信内打开速度和稳定性可能受影响。
